# 开发日志

## 2026-04-03 项目初始化

### 项目目标
构建高性能的A股多因子回测框架，专门针对"月度调仓"和"基本面因子"设计，完全规避"未来函数"。

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    Main Entry (main.py)                  │
├─────────────────────────────────────────────────────────┤
│  DataFetcher  │  DataManager  │  FactorProcessor        │
│  (数据获取)   │  (数据管理)    │  (因子计算)             │
├─────────────────────────────────────────────────────────┤
│              Backtester (回测引擎)                       │
│  - MonthlyRebalanceEngine                                │
│  - PerformanceAnalyzer                                   │
├─────────────────────────────────────────────────────────┤
│           ReportGenerator (报告生成)                     │
└─────────────────────────────────────────────────────────┘
                      ↓
              DuckDB 数据库
```

### 核心模块

1. **data_fetcher.py** - 数据获取
   - 使用 AkShare 获取数据（免费、无需API Key）
   - 支持增量更新
   - 处理复权（前复权/后复权）
   - 标记停牌期

2. **data_manager.py** - 数据管理
   - DuckDB 数据库管理
   - 交易日历查询
   - 可交易股票过滤（ST、停牌、涨跌停、上市天数）
   - 数据校验

3. **factor_processor.py** - 因子处理
   - 基础因子：动量、波动率、市值、反转、流动性
   - 预处理流水线：MAD去极值 → 中性化 → 标准化
   - IC计算

4. **backtester.py** - 回测引擎
   - 月度调仓
   - 分层回测
   - 绩效计算

5. **report_generator.py** - 报告生成
   - 净值曲线图
   - 分组收益对比
   - 多空收益分析
   - 文本报告

### 关键设计决策

#### 1. 数据库选择：DuckDB
- 原因：列式存储、SQL支持、单文件、零配置
- 对比：SQLite（行式，OLAP性能差）、Parquet（无SQL）
- 性能：千万级行数查询 <1s

#### 2. 复权处理
- 存储：原始价格 + 复权因子
- 计算：前复权价格 = 原始价格 × 复权因子
- 灵活：可切换前复权/后复权/不复权

#### 3. 未来函数规避
- 财务数据：使用 `ann_date`（公告日）
- 调仓时机：T日计算，T+1日执行
- 价格数据：使用T-1日及之前数据

#### 4. 因子预处理顺序
1. 去极值（MAD）- 防止极端值影响
2. 行业中性化 - 消除行业偏差
3. 市值中性化 - 消除规模偏差
4. 标准化（Z-Score）- 统一量纲

### 使用方法

```bash
# 安装依赖
pip install -r requirements.txt

# 运行回测（Windows）
run.bat

# 或直接运行
python main.py
```

### 注意事项

1. 首次运行会下载沪深300成分股数据，耗时约10-30分钟
2. AkShare请求有频率限制，间隔设为0.3秒
3. 报告保存在 `_output/` 目录
4. 日志保存在 `logs/` 目录

### 性能指标

- 数据下载：约 300 只股票，3 年数据，约 5-15 分钟
- 因子计算：每个调仓日约 1-3 秒
- 回测执行：约 1-5 秒
- 报告生成：约 5-10 秒
