# A股数据生产级别验证任务设计

## 概述

本验证套件旨在系统性验证 stock_data_project 项目获取的 A 股市场历史交易数据是否满足**生产级别的量化选股和回测要求**。

---

## 验证框架

### 验证维度

| 维度 | 目标 | 核心关注点 |
|------|------|-----------|
| **数据完整性** | 确保数据覆盖全面 | 股票池、时间范围、断层检测 |
| **数据准确性** | 确保数据正确可靠 | OHLC校验、复权因子、量价关系 |
| **PIT查询正确性** | 消除未来函数 | 历史股票池、指数成分股、公告日期对齐 |
| **幸存者偏差处理** | 消除回测偏差 | 退市股数据、ST状态历史 |
| **交易规则模拟** | 准确模拟实盘 | T+1冻结、涨跌停、100股取整 |
| **回测引擎端到端** | 验证完整链路 | 信号→执行→结算→绩效 |
| **因子计算准确性** | 确保因子可信 | ROE/PB/PE等关键因子 |
| **性能压力测试** | 确保生产可用 | 大规模回测、并发查询 |

---

## 详细验证任务清单

### 1. 数据完整性验证

#### 1.1 股票池覆盖度
- **验证方法**: 使用 PIT 查询获取历史各时点的可交易股票数量
- **期望结果**: 4000-6000 只股票（A股市场规模）
- **关键时点**: 2022-06-30, 2023-06-30, 2024-06-30
- **判定标准**: 通过 / 警告 / 失败

#### 1.2 时间范围覆盖
- **验证方法**: 查询数据库中实际数据的日期范围
- **期望结果**: 至少覆盖3-5年历史数据
- **判定标准**: 通过 / 失败

#### 1.3 数据断层检测
- **验证方法**: 抽样检测股票的时间序列完整性
- **关注点**: 停牌期间是否有数据、是否有跳跃
- **判定标准**: 无断层 / 轻微断层 / 严重断层

#### 1.4 年均交易天数
- **验证方法**: 统计每年的实际交易天数
- **期望结果**: 每年240-250个交易日
- **判定标准**: 通过 / 警告 / 失败

#### 1.5 股票数量稳定性
- **验证方法**: 按月统计有数据的股票数量
- **关注点**: 波动是否合理（退市股处理后）
- **判定标准**: 波动<20% / 波动>20%

---

### 2. 数据准确性验证

#### 2.1 OHLC一致性
- **验证方法**: 检查 high < open/close/low 等违规情况
- **期望结果**: 违规率 < 0.1%
- **判定标准**: 通过 / 警告 / 失败

#### 2.2 复权因子准确性
- **验证方法**: 检查复权因子是否 >= 0 且变化平滑
- **关注点**: 是否有大幅跳变（>50%）
- **判定标准**: 无异常 / 轻微异常 / 严重异常

#### 2.3 量价关系
- **验证方法**: 检查成交量为0但价格有变化的情况
- **期望结果**: 停牌误记录率 < 0.1%
- **判定标准**: 通过 / 警告 / 失败

#### 2.4 涨跌幅一致性
- **验证方法**: 验证 pct_chg 与 (close-pre_close)/pre_close 的一致性
- **期望结果**: 不一致率 < 1%
- **判定标准**: 通过 / 警告 / 失败

#### 2.5 价格极值
- **验证方法**: 检查价格是否在合理范围 (0.01 ~ 10000)
- **判定标准**: 无异常 / 存在极端值

---

### 3. PIT查询验证

#### 3.1 PIT股票池查询
- **验证方法**: 使用 get_active_stocks() 查询历史时点
- **关注点**: 是否正确过滤上市/退市日期
- **判定标准**: 查询成功且数量合理 / 查询失败

#### 3.2 PIT指数成分股查询
- **验证方法**: 使用 get_index_constituents() 查询历史成分股
- **关注点**: 沪深300等主要指数的成分股数量 (280-320)
- **判定标准**: 查询成功且数量合理 / 查询失败

#### 3.3 公告日期对齐
- **验证方法**: 检查 financial_data 表的 ann_date
- **关注点**: 是否存在未来公告日期（未来函数）
- **判定标准**: 无未来数据 / 存在未来数据

#### 3.4 eff_date生效日期逻辑
- **验证方法**: 检查 stock_basic_history 表的日期逻辑
- **关注点**: eff_date <= end_date 等约束
- **判定标准**: 逻辑正确 / 存在异常

---

### 4. 幸存者偏差验证

#### 4.1 退市股覆盖
- **验证方法**: 统计已退市股票数量
- **期望结果**: 历史退市股 >= 100 只
- **判定标准**: 充足 / 不足 / 严重缺失

#### 4.2 ST状态历史
- **验证方法**: 检查 st_status_history 表
- **期望结果**: 有ST历史的股票 >= 100 只
- **判定标准**: 充足 / 不足 / 无数据

#### 4.3 退市日期准确性
- **验证方法**: 检查退市股是否有退市日期
- **关注点**: is_delisted=TRUE 但 delist_date=NULL
- **判定标准**: 完整 / 存在缺失

#### 4.4 股票生命周期覆盖
- **验证方法**: 统计有完整生命周期记录的股票
- **关注点**: 是否有多次状态变更记录
- **判定标准**: >= 100只 / < 100只

---

### 5. 交易规则验证

#### 5.1 T+1冻结逻辑
- **验证方法**: 模拟买入操作，检查持仓的 available 字段
- **期望结果**: 当日买入股票，available=0
- **判定标准**: 正确 / 错误 / 无法验证

#### 5.2 涨跌停检测
- **验证方法**: 测试各板块股票的涨跌停阈值
- **期望结果**:
  - 主板(60xxxx, 00xxxx): ±10%
  - 创业板(300xxx): ±20%
  - 科创板(688xxx): ±20%
- **判定标准**: 全部正确 / 部分错误 / 全部错误

#### 5.3 板块涨跌停规则
- **验证方法**: 测试 detect_board() 和 get_price_limit()
- **判定标准**: 板块识别正确 + 涨跌停正确 / 任一错误

#### 5.4 100股取整
- **验证方法**: 测试取整逻辑
- **期望结果**: 123→100, 150→100, 200→200
- **判定标准**: 正确 / 错误

---

### 6. 回测引擎验证

#### 6.1 简单回测运行
- **验证方法**: 运行1-3个月的回测
- **关注点**: 是否能正常运行并产生结果
- **判定标准**: 运行成功且有结果 / 运行失败

#### 6.2 权益曲线完整性
- **验证方法**: 检查回测结果的 equity_curve
- **关注点**: 非负性、单调性
- **判定标准**: 完整且正确 / 无数据 / 存在问题

#### 6.3 持仓结算
- **验证方法**: 模拟交易后调用 settle()
- **关注点**: 持仓是否正确记录
- **判定标准**: 正确 / 错误

#### 6.4 换手率数据
- **验证方法**: 检查 turnover 字段覆盖率
- **期望结果**: 覆盖率 > 90%
- **判定标准**: 通过 / 警告 / 失败

---

### 7. 因子计算验证

#### 7.1 财务数据覆盖
- **验证方法**: 检查 financial_data 表记录数
- **期望结果**: 有财务数据且覆盖主要股票
- **判定标准**: 有数据 / 无数据

#### 7.2 因子值范围
- **验证方法**: 检查 ROE、PB 等因子值是否在合理范围
- **期望结果**: ROE (-100%~100%), PB (0.1~50) 等
- **判定标准**: 合理 / 存在异常

#### 7.3 因子计算逻辑
- **验证方法**: 验证因子间的关系一致性
- **关注点**: 净利润 = 营业收入 × 净利率
- **判定标准**: 一致 / 存在不一致

---

### 8. 性能压力测试

#### 8.1 单次查询性能
- **验证方法**: 查询单只股票全年数据
- **期望结果**: < 100ms
- **判定标准**: 快 (<100ms) / 中 (100ms-1s) / 慢 (>1s)

#### 8.2 批量查询性能
- **验证方法**: 查询100只股票的某日数据
- **期望结果**: < 1s
- **判定标准**: 快 / 中 / 慢

#### 8.3 大规模回测性能
- **验证方法**: 运行1个月的回测
- **期望结果**: < 10s
- **判定标准**: 快 / 中 / 慢
